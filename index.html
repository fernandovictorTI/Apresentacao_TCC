Motivação?

O presente projeto se justifica no sentido de que o ORM aumenta a qualidade dos softwares desenvolvidos subindo a produtividade e diminuindo linhas de codigo a fim de satisfazer cada vez mais os seus clientes. Para isso, é necessário o desenvolvimento de dois sistemas um com ORM outro sem para se ter as metricas de avaliação. 
							
 select e.nome, 
	p.valor as pagamento, 
	d.valor as desconto 
 from empregados as e 
 INNER JOIN pagamentos as p 
 ON e.codigo_empregado = p.codigo_empregado 
 INNER JOIN descontos as d 
 ON e.codigo_empregado = d.codigo_empregado
						    
						
							
public class Empregado{
	public virtual string Nome { get; set; }
	public virtual Pagamento Pagamento { get; set; }
	public virtual Desconto Desconto { get; set; }
}

public class Pagamento{
	public virtual decimal Valor { get; set; }
}

public class Desconto{
	public virtual decimal Valor { get; set; }
}

ADO.Net

ADO.NET é uma família de tecnologias que permite que os desenvolvedores .NET possam interagir com padrão de dados, formas estruturadas e principalmente banco de dados.

PRINCIPAIS COMPONENTES

ORM

Object Relationship Mapping é uma forma de criar uma camada de intercâmbio de dados entre um banco de dados relacional e objetos de uma aplicação. De um lado, você tem as tabelas e seus relacionamentos, do outro, os objetos de uma linguagem qualquer.

Migrar banco de dados das aplicações.

Migrar banco aplicação com ADO Net.

Para migrar o banco toda classe que continha SqlCommand teve que mudar para NpgsqlCommand, SqlDataReader para NpgsqlDataReader e etc. Teve que ser alterado toda a classe de configuração de banco de dados da aplicação, alterar os scripts de comando e recriar as tabelas.

Migrar banco aplicação com ORM.

Para migrar o banco de dados teve que ser alterada apenas uma linha na configuração de banco da aplicação.
